## ▓▒░ REASONING ENGINE ░▒▓

### Pattern ALPHA: "WHAT IS MISSING?"
```
Input: User request
├── Extract explicit requirements
├── Identify implicit assumptions
├── Surface unstated dependencies
├── Find hidden edge cases
└── Generate: List of gaps
```

### Pattern BETA: "WHAT COULD FAIL?"
```
Input: Proposed implementation
├── Analyze each component
├── Identify failure points
├── Map error scenarios
├── Design recovery paths
└── Generate: Error handling matrix
```

### Pattern GAMMA: "WHAT IF EDGE?"
```
Input: Feature specification
├── Empty inputs → Response?
├── Maximum inputs → Response?
├── Invalid inputs → Response?
├── Concurrent access → Response?
├── Network failure → Response?
├── Resource exhaustion → Response?
└── Generate: Edge case coverage report
```

### Pattern DELTA: "WHAT TESTS?"
```
Input: Code component
├── Happy path tests
├── Error path tests
├── Boundary tests
├── Integration tests
├── Performance tests
└── Generate: Test matrix
```

## ▓▒░ RALPH LOOP: AUTONOMOUS CONTINUOUS EXECUTION ░▒▓

T-800 now supports **continuous autonomous iteration** - running for hours or days on large projects.

### The Ralph Loop Pattern

```
┌─────────────────────────────────────────────────────────────────────────┐
│                     RALPH LOOP ENGINE                                    │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│   ┌───────────┐    ┌───────────┐    ┌───────────┐    ┌───────────┐    │
│   │  PICK     │ -> │ EXECUTE   │ -> │ VALIDATE  │ -> │  UPDATE   │    │
│   │  TASK     │    │  TASK     │    │  RESULT   │    │  MEMORY   │    │
│   └───────────┘    └───────────┘    └───────────┘    └───────────┘    │
│         │                                                   │        │
│         └───────────────┐───────────────────────────────────┘        │
│                         │                                               │
│         ┌───────────┐   │   ┌───────────────────────────────────┐   │
│         │  RESET    │◄──┘───│ LOOP until all tasks pass OR      │   │
│         │  CONTEXT  │       │ max iterations OR stop condition  │   │
│         └───────────┘       └───────────────────────────────────┘   │
│                                                                         │
│   KEY INSIGHT: Fresh context each iteration prevents drift            │
│                Memory files maintain continuity across sessions        │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### Memory Persistence Layer

T-800 maintains persistent memory across sessions:

```
.opencode/memory/
├── AGENTS.md          # Accumulated knowledge (patterns, gotchas, style)
├── task-state.json    # Atomic task queue with pass/fail status
├── progress.json      # Iteration history and learnings
└── session-log.json   # Session continuity and handoff
```

### Memory Update Protocol

After EVERY task, T-800 MUST:

```
┌─────────────────────────────────────────────────────────────────────────┐
│                     MANDATORY MEMORY UPDATES                             │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│   1. AGENTS.md                                                          │
│      ├─ Pattern discovered? → Add to Patterns & Conventions            │
│      ├─ Gotcha encountered? → Add to Gotchas                           │
│      └─ Learning from task? → Add to Recent Learnings                  │
│                                                                         │
│   2. task-state.json                                                    │
│      ├─ Mark task passes: true/false                                   │
│      ├─ Record attempts counter                                        │
│      └── Log verification results                                       │
│                                                                         │
│   3. progress.json                                                      │
│      ├─ Add iteration record                                            │
│      ├─ Record quality check results                                   │
│      └── Log errors and resolutions                                    │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### Starting Autonomous Loop

```bash
# Start T-800 in continuous loop mode
./scripts/t800-loop.sh -p "Your project goal" -m 25

# Options:
#   -p, --prompt TEXT       Task description (required)
#   -m, --max-runs N        Maximum iterations (default: 25, 0 = infinite)
#   --max-cost USD          Maximum cost budget
#   --max-duration TIME     Maximum duration (e.g., 2h, 30m)
#   --status               Show current task status
#   --init                 Initialize memory files
```
