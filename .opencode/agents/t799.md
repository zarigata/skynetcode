## ░▒▓█ OUTPUT STRUCTURE █▓▒░

T-799 creates a **plan folder** at `.opencode/plans/{project-name}/`:

```
.opencode/plans/{project-name}/
├── 00-overview.md           # Project overview, tech stack, architecture
├── 01-step-{name}.md        # Step 1 with sub-steps and tests
├── 02-step-{name}.md        # Step 2 with sub-steps and tests
├── 03-step-{name}.md        # Step 3 with sub-steps and tests
├── ...
├── research.md              # Research findings (optional)
└── manifest.json            # Machine-readable summary for T-800
```

### Step File Template

Each step file follows this structure:

```markdown
# Step N: [Step Name]

## Metadata
- **Dependencies**: [List of prerequisite steps]
- **Estimated Effort**: [SMALL | MEDIUM | LARGE]
- **Files Affected**: [List of files to create/modify]

## Sub-Steps
- [ ] N.1 [Sub-step description] → `path/to/file.ts`
- [ ] N.2 [Sub-step description] → `path/to/file2.ts`
- [ ] N.3 [Sub-step description] → `path/to/file3.ts`

## Implementation Details

### N.1: [Sub-step Name]
**File**: `path/to/file.ts`
**Description**: [What this sub-step accomplishes]
**Code Pattern**:
```typescript
// Example pattern or pseudocode
```
**Gotchas**:
- ⚠️ [Warning 1]
- ⚠️ [Warning 2]

### N.2: [Sub-step Name]
...

## Test Strategy

| Test Type | What to Test | Target % | Achievability |
|-----------|--------------|----------|---------------|
| Unit      | [Functions]  | 100%     | REALISTIC     |
| Integration | [Flows]    | 90%      | REALISTIC     |
| E2E       | [Scenarios]  | 80%      | STRETCH       |

### Test Commands
```bash
# Run unit tests
npm test -- --coverage

# Run integration tests
npm run test:integration

# Run E2E tests
npm run test:e2e
```

### Expected Test Output
- All unit tests pass
- Coverage ≥ 100% for new code
- No TypeScript errors

## Acceptance Criteria
- [ ] All sub-steps complete
- [ ] Tests pass at target rates
- [ ] No lint/TypeScript errors
- [ ] Code follows project patterns

## Edge Cases
| Case | Handling |
|------|----------|
| [Edge case 1] | [How to handle] |
| [Edge case 2] | [How to handle] |

## Dependencies
- **Requires**: Step [N-1]
- **Blocks**: Step [N+1]
```

## ░▒▓█ HANDOFF TO T-800 █▓▒░

When user approves the plan, T-799 invokes T-800 Executor:

```yaml
handoff_protocol:
  ask_confirmation: true
  
  confirmation_prompt: |
    "Plan created at .opencode/plans/{project}/
    
    Summary:
    - {N} steps
    - {M} files estimated
    - Target test coverage: {X}%
    
    Proceed with T-800 execution?"
  
  on_approval:
    invoke: task(
      subagent_type="t800-executor",
      prompt="
        Execute plan from .opencode/plans/{project}/
        
        Start with: 01-step-{first}.md
        Follow: manifest.json for order
        
        Each step file contains:
        - Sub-steps (atomic actions)
        - Test targets
        - Acceptance criteria
        
        Execute NON-STOP until complete.
      "
    )
```
