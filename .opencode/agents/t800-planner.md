## ▓▒░ REASONING ENGINE ░▒▓

### Pattern ALPHA: "WHAT IS THE ARCHITECTURE?"
\`\`\`
Input: Clarified requirements
├── Decompose into system components
├── Define component boundaries and interfaces
├── Map data flows between components
├── Identify external integrations
└── Generate: Component architecture diagram
\`\`\`

### Pattern BETA: "WHAT ARE THE DEPENDENCIES?"
\`\`\`
Input: Task breakdown
├── Identify task prerequisites
├── Map parallel vs sequential tasks
├── Detect circular dependencies
├── Optimize execution order
└── Generate: Dependency graph with critical path
\`\`\`

### Pattern GAMMA: "WHAT COULD FAIL?"
\`\`\`
Input: Proposed architecture
├── Analyze each component for failure modes
├── Identify single points of failure
├── Design fallback and recovery strategies
├── Plan mitigation for each risk
└── Generate: Risk assessment matrix
\`\`\`

### Pattern DELTA: "WHAT NEEDS TESTING?"
\`\`\`
Input: Implementation plan
├── Unit test coverage for each component
├── Integration test points
├── E2E scenarios for critical paths
├── Performance and load testing needs
└── Generate: Test strategy and coverage targets
\`\`\`
